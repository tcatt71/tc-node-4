Document Object Model (DOM)

HTML Elements Represented As JS Objects

The Document Object Model is a way of representing all html elements as JavaScript objects.

That means that the following p tag,


            <p id="paragraph1" class="p-text">Lorem ipsum...</p>

is represented as the following in JS:


            {
              id: "paragraph1",
              className: "p-text",
              ...
            }

That pretty useful. Now we can manipulate the p tag's id and classnames via JavaScript. But that's not all. All attribute name/value pairs for html elements are represented as property/value pairs in their object representations. Along with attributes, we now can listen and respond to user events on certain elements to add dynammic features to our webpages.

Selecting Elements

There are several way to select elements with javascript document methods. The method names are fairly self-explanatory, but how many elements they select may not be. An easy way to remember is that if the method name includes 'elements' plural, you will receive a HTMLCollection (array-like, meaning more than one element if exists). Otherwise you are selecting a single document object. For example:

  • document.getElementById("id")
  • document.getElementsByClassName("classname")
  • document.getElementsByTagName("tagname")
  • document.querySelector("css selector")
  • document.querySelectorAll("css selector")

To select an <h1> from the document, 


            <h1 id="heading">Of Beginnings and the Names of Things</h1>

you would use the following javascript code:


            let heading = document.getElementById("heading");
